Optimizations implemented

          

            

              
✓ Opt 1: Keyword routing (saves ~4s)

              
check_routing was calling LLM (~4s) to decide "needs_tax?" — a purely keyword-based decision. Replaced with frozenset matching in law_agent/graph.py.

            

            

              
✓ Opt 2: Registry discover cache (saves ~40ms)

              
In-process _discover_cache dict in common/registry_client.py. Task→endpoint mappings are stable per run.

            

            

              
✓ Opt 3: AgentCard cache (saves ~30ms)

              
In-process _card_cache dict in common/a2a_client.py. AgentCard JSON is static while service is running.

            

            

              
○ Future: Streaming partial responses

              
LLM inference (~16s) is the real bottleneck. Streaming lets users see results as they arrive instead of waiting for the full pipeline.

            

          

        

      

    

  


  
  

    
How to Run

    

      

        
1. Start all services

        
# Linux/macOS
./start_all.sh

# Windows PowerShell
./start_all.ps1

# Wait ~10s for all agents
# to register with Registry

      

      

        
2. Start the proxy

        
# Activate venv first:
.venv\Scripts\activate  # Win
source .venv/bin/activate  # Mac

python demo_proxy.py
# Starts on :8765

# Or use uv:
uv run python demo_proxy.py

      

      

        
3. Test

        
# Open this file in browser:
agent_interaction_demo.html

# Or via proxy (auto-serves HTML):
http://localhost:8765

# CLI test:
python test_client.py
